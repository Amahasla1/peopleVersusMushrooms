# Рабочие заметки по mushroomsArmy

## Ближайшая цель
- Перевести ключевой игровой поток на сокеты.
- На клиенте показать карту, юнитов и несколько целей.
- Запустить базовое перемещение юнитов к целям.

## Приоритетные задачи
1. Добавить интервалы обновления: `200` секунд.
2. Вынести неструктурированную логику в отдельный модуль/слой.
3. Для армии вызывать метод `update`, передавать:
     - `armyGuid` (совпадает с `playerGuid`),
     - данные состояния.
4. Спроектировать блок армии, в котором юнит выбирает цель, куда бежать.
5. К следующей паре подготовить рабочую демо-версию.
6. На клиенте отобразить:
     - карту,
     - юнитов,
     - несколько целей.
7. Реализовать движение юнитов к целям.
8. Добавить/дописать клиентскую часть юнита.

## Клиент
- Пересмотреть `useState`/`useRef` там, где не нужен ререндер.

## Сокеты и архитектура
- Работать только через сокеты в ключевых сценариях.
- Сокеты двусторонние: сервер может отправлять события клиенту без запроса.
- Клиентские запросы отправлять через `emit`.
- Перенести сокет-логику ближе к сущности пользователя (`User`/`UserManager`).
- Обеспечить единообразные названия триггеров/событий.
- В запросах менеджеров передавать `guid` пользователя.

## Сервер
- В `server` проверить и почистить устаревшие обработчики ошибок.
- Получать пользователя по `guid` (`GET_USER_BY_GUID`).
- Добавить/проверить модуль армии на уровне `application`.

## Технический долг
- Убрать `ts-ignore`.
- Проверить измененные модули: `server`, `app`, `login`, `types`, `mediator`, `useMediator`, `config`, `index`.

## Минимум к демонстрации
1. Отрисовка карты.
2. Отрисовка юнитов и целей.
3. Движение юнитов к цели.
