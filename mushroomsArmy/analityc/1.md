# Рабочие заметки по mushroomsArmy

## Обязательные принципы разработки
- Обмен данными между серверными модулями выполнять только через `mediator`.
- Использовать БД минимально: основное состояние хранить в активных записях (in-memory).

## Общая архитектура
- Backend: Node.js + TypeScript + Express + Socket.IO.
- Frontend: React + TypeScript + Socket.IO-client.
- База данных: SQLite3.
- Стилизация: CSS.

## Правила по mediator
- `Mediator` — единая точка обмена данными между серверными модулями.
- Прямые вызовы методов между менеджерами/модулями запрещены, только события и триггеры через `mediator`.
- Все имена событий и триггеров должны быть единообразными и храниться в конфиге.
- Для каждого события/триггера фиксировать контракт payload (какие поля приходят и что возвращается).
- Логику бизнес-процессов держать в модулях, а `mediator` использовать только как шину обмена и связывания.
